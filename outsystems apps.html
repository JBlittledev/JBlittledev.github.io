Field Flow

  This project's scope was to create an application so that our clients could book sports fields in differentlocations and at different times. It was a project that I embraced from the start and in which I took on new tasks unrelated to development. Mymain concerns during the initiation week were, together with the project manager, to meet with the client and runa set of questions to better understand their needs. To clarify the process, I worked on process diagrams and created personas. I also designed wireframes, theproject architecture, and the data model to ensure a solid foundation for what was to come.

Project Tools and Technologies

  • Service Studio:IDE used for development.
  • Agile:Applied agile methodologies and drove ceremonies to ensure a well-directed and organized workflow throughout the project.
  • JIRA: Used for developing and managing the backlog with user stories and sprint planning.
  • REST Consumed REST APIs from IPMA to obtain weather forecasts for different locations.
  • Architecture Canvas Applied the three-layer canvas to design the project architecture.
  • Gherkin: Used Gherkin syntax as one of the criteria to write user stories, aligning them with the standards of the Definition of Done (DoD).
  • Balsamiq: Used for designing wireframes.
  • Canvas: A tool used to design personas and process diagrams.
